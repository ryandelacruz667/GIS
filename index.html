B. Create a new GRASS database

1. Running GRASS GIS in Windows

Click Windows Start menu, and type grass and Windows will autofill the rest of the GRASS GIS 7.8.7 as shown below.

Figure 1. Opening GRASS GIS application in Windows 10

2. Define GRASS directory, location, and mapset

  1. Directory – simply a folder that stores grass database
  2. Location – a subfolder of Directory that must have the same Coordinate Reference System (CRS)
  3. Mapset – a user folder, esp. useful in a shared network of computers. The default is the PERMANENT mapset which only the creator can edit. Other users can read (but not write/modify) maps from the PERMANENT mapset. Each network use must create their own mapset

Figure 2. GRASS GIS startup windows

a. Create or select a GRASS database directory

Click the Browse button in Figure 3 to choose or create a new GRASS database directory

Figure 3. Create a GRASS GIS database directory

b. Create or select a GRASS database location

Enter the folder name of your new GRASS database location, then click the Next button.

Figure 4. Create or select a GRASS GIS location

c. Selecting a location's CRS

There are several ways to select your preferred CRS, but this is the easiest if you know the EPSG code of your preferred CRS. Select Select EPSG code of spatial reference system and click the Next button.

Figure 5. Create or select a GRASS GIS location

d. Enter the EPSG code of your preferred CRS

Enter 32651 to use UTM 51N. The set coordinate reference system (CRS) upon initialization cannot be modifed again in the future.

Figure 6. Enter the EPSG code

e. Select a datum transformation

Just click the OK button to select the default WGS 84 datum

Figure 7. Select the WGS84 datum

Verify the parameters of your newly created GIS database and CRS projection. Click the Finish button if you are satisfied.

Figure 8. Select the WGS84 datum

g. Run the GIS application

An empty GRASS database with your choosen CRS has been created. Click the Start GRASS session button to start developing maps.

Figure 9. Ready to run a GRASS GIS session

C. GRASS GIS Data Processing

GRASS GIS application has 3 default windows upon opening. These include the (1) GRASS console, (2) GRASS GIS Map Display Layer, and the (3) GRASS Map Display Layer Manager as shown in Figure 10.

Figure 10. GRASS GIS application windows

1. Import a raster image using r.in.gdal

The newly created GRASS database is still empty. The only data added in the previous section is the CRS for the map location.

r.in.gdal input=D:/Jun_turnover_files/DEM/dem_region8.tif output=dem8 --o

g.message "check to see if dem8 raster exists"
g.list raster
The r.in.gdal command has 2 parameters and the 3rd called switch:
  1. input paramater: name of the input digital elevation model
  2. output parameter: name of the output raster
  3. --o switch: overwrites the output name if the raster already exists

2. Set the environmental settings

Run the following code in GRASS GIS console

g.region -p

The g.region -p will show that the area of interest is only 1x1 pixel. GRASS GIS ignores any data outside the area of interest. We should change it to follow the extent of the imported raster.

g.region raster=dem8 -p

The area of interest should now coincide with the extent of dem8 raster. The -p switch will display in the console the new area of interest as follows:

3. Import a vector data for region 8 provinces

v.in.ogr input=D:/gisclass/samarisland.geojson output=samarisland --o

g.message "check if the output vector is created"
g.list vector

4. Erase columns from the attribute table of imported vector data

v.db.dropcolumn map=samarisland columns=ID_0,ISO,ID_1,NAME_0,VARNAME_1,NL_NAME_1,HASC_1,CC_1,TYPE_1,ENGTYPE_1,VALIDFR_1,VALIDTO_1,REMARKS_1

g.message "check column names"
db.columns table=samarisland
g.message "the remaining columns are cat, NAME_1, Shape_Leng, Shape_Area"

5. Add a new column and assign a value of 1

g.message "add an column name imask"
v.db.addcolumn map=samarisland columns="imask integer"
g.message "assign a value of 1 to imask column"
v.db.update map=samarisland column=imask value=1

6. Convert vector to raster

The code below will create a raster with a value of 1 for the 3 provinces of Samar, the rest will have a value of zero

v.to.rast input=samarisland type=area output=samisland use=attr attribute_column=imask value=1 --o

7. Change the region area of interest to the non-zero extent of the samarisland

The code below will reduce the extent of the area of interest in our analysis. The g.region -p will display their difference before and after running the g.region zoom=samisland code

g.message "display region extent"
g.region -p

g.message "reduce the extent of the region area of interest"
g.region zoom=samisland

g.message "display region again to see changes in number of rows and cols"
g.region -p

8. Limit processing only within the island of Samar

The rmask module will limit data processing only within the non-zero pixel of the samisland raster. The rest are ignored.

r.mask raster=samisland

9. Delineate a watershed basin boundary

r.watershed elevation=dem8 threshold=50000 basin=d.basin stream=d.stream --o

The r.watershed module above delineates a watershed basin boundary and stream network from a digital elevation model (DEM). Use dem8 as the input DEM, threshold or a minimum 50,000 pixels (resolution is 30x30 meter) to be considered a watershed. Any sub-watershed that has less than 50,000 pixels are ignored, leading to empty pixels along the coastal areas. The outputs are:

  1. basin=d.basin – where d.basin is a raster of watershed basin delineation
  2. d.stream – a raster file of “synthetic” network of rivers and stream.
  3. “--o” switch allows overwriting of existing output rasters

10. Delineate a synthetic river/stream

The r.watershed command above also delineates a raster of synthetic river network. The code below converts it from raster to vector format.

g.message "1. r.thin decreases the width rivers/streams into 1 pixel wide only,
while the length is unlimited"
r.thin --o input=d.stream output=stream_thin

g.message "2. raster to vector conversion"
r.to.vect --o type=line input=stream_thin output=vstream

The vstream is the output network of streams/rivers in a vector format.

11. Generate a landform map

r.geomorphon elevation=dem8 forms=rgeomorph flat=5
r.category rgeomorph

The output of the r.category rgeomorph is as follows:

1       flat
2       summit
3       ridge
4       shoulder
5       spur
6       slope
7       hollow
8       footslope
9       valley
10      depression

Click r.geomorphon for more details about the geomorphology map output.

12. Remove mask of the current GRASS location

The GRASS location mask set using the r.mask module is saved, and will remain until you remove it using

r.mask -r

D. Opening GRASS database in QGISTM

Finally, display the GRASS output in QGIS in order to be able to compose a beautiful map composition. Although GRASS GIS can display a map, QGIS is better in composing maps.

1. Open a new project in QGIS

Run the QGIS application and create a blank project by double-clicking the New Empty Project highlighted in blue in Figure 11.

Figure 11. GRASS GIS application windows

2. Change a project's CRS

Notice however that the default CRS upon opening QGIS is WGS84 with EPSG 4326. To change your project DRC, select the EPSG 4326 on bottom-right of QGIS encircled with red in Figure 12, which opens the Project Properties | CRS in Figure 13.

Figure 12. GRASS GIS application windows

The Filter box with highlighted with a red circle in Figure 13, enter 32651. Finally, click the WGS 84 / UTM 51N highlight in blue and encircled with yellow in Figure 13. Click the OK button to close the Properties window.

Figure 13. GRASS GIS application windows

Next, click the pulldown menu Plugins ==> GRASS ==> Open Mapset as shown in Figure 14.

Figure 15. GRASS GIS application windows

4. Load map layers from GRASS database

Although GRASS has a good display of map layers, QGIS is better in composing a map for printing or digital display. It is therefore recommended to generate map layers in GRASS GIS, and then layout a map composition in QGIS.

Use the Browser panel (see Figure 17) to search for specific map layers from GRASS database. In my example, I save my GRASS database in D:\Session6, my Location in sessionLocation, and PERMANENT as my Mapset. The Browser panel marks a GRASS database Session6 with a database icon, the Location D:\Session6\sessionLocation with a GRASS icon, the Mapset D:\Session6\sessionLocation\PERMANET with a mapset icon, and the different layers, whether raster, polygon, or line lyers with appropriate icons to easily classify them as shown in Figure 17.

Drag the vgeomorph raster to the Layers panel. The layer is vgeomorph layer is not yet visible so we have to change its color in the next step.

Figure 17. The Browser panel displays the location and layer names from a GRASS GIS database. The Layers panel shows the map layers loaded in the current QGIS project.
